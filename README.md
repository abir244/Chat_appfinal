# Kulikéun Chat - A Modern Real-time Messenger

Kulikéun is a feature-rich, real-time chat application built with Flutter and Supabase. It features a stunning, modern UI inspired by the "Kulikéun" design concept, combining sleek dark-mode aesthetics with fluid glassmorphism elements. Beyond messaging, it integrates live news and travel photo feeds to create an engaging user experience.

## Features

- **Real-time Messaging**: Instant one-on-one text and media (image/video) messaging.
- **Message Requests**: A full request system (send, accept, reject) to prevent spam.
- **Social Features**: Unfriend users and manage your contact list.
- **User Profiles**: Customizable user profiles with avatar image uploads.
- **User Presence**: See when users are `Online` or `Offline` with real-time status indicators.
- **Dynamic Feeds**: 
    - **Global News**: A dedicated tab showing the latest headlines from around the world via the GNews API.
    - **Travel Explore**: A beautiful, image-heavy feed of travel photos from the Unsplash API.
- **Modern UI/UX**: 
    - Custom-designed "Kulikéun" theme.
    - Reusable glassmorphism widgets.
    - Floating navigation bar with a central "Magic" button.
- **Backend with Supabase**:
    - Secure user authentication (Email/Password).
    - Real-time database for chat and user presence.
    - File storage for avatars and chat media.

## Tech Stack

- **Frontend**: Flutter
- **Backend**: Supabase (Auth, Realtime Database, Storage)
- **State Management**: Riverpod
- **APIs**: GNews, Unsplash
- **Architecture**: Clean, with a modular structure (Data, Presentation, Core).

## Setup & Installation

To get the project running locally, follow these steps:

### 1. Flutter Environment
   Ensure you have the Flutter SDK installed and an emulator/device ready.

### 2. Supabase Setup
   - Create a new project at [supabase.com](https://supabase.com).
   - In your project dashboard, go to the **SQL Editor** and run all the `.sql` scripts located in the `/supabase` directory of this project to create the necessary tables (`users`, `messages`, `groups`, etc.) and database functions (`get_conversations`).
   - Go to the **Storage** tab and create two **public** buckets:
     - `avatars`
     - `chat_files`
   - Go to the **Authentication** > **Providers** section and enable the **Email** provider. For development, it's recommended to turn **OFF** "Confirm email".
   - Go to **Project Settings** > **API**. Copy your `URL` and `anon` `public` key.

### 3. Configure API Keys
   - Open the file `lib/core/constants/app_constants.dart`.
   - Paste your Supabase URL and anon key:
     ```dart
     static const supabaseUrl = 'YOUR_SUPABASE_URL';
     static const supabaseAnonKey = 'YOUR_SUPABASE_ANON_KEY';
     ```
   - Get a free API key from [gnews.io](https://gnews.io/) and add it:
     ```dart
     static const gNewsApiKey = 'YOUR_GNEWS_API_KEY';
     ```
   - The Unsplash key is already included for the demo.

### 4. Run the App
   - Open a terminal in the project root and run:
     ```sh
     flutter pub get
     flutter run
     ```

## Project Structure

The project is organized into three main layers:

- `lib/core`: Contains the global theme (`app_theme.dart`), API constants, and dependency injection providers.
- `lib/data`: Includes models (e.g., `user_model.dart`) and repositories that handle communication with the backend (e.g., `chat_repository.dart`).
- `lib/presentation`: Contains all the UI-related files, separated by feature (e.g., `home`, `chat`, `auth`), along with reusable widgets (`kuli_widgets.dart`).
